# AI 协作约定（跨 IDE 通用）

在 Cursor、Codex、Copilot 或其它 AI 助手中，先遵守本约定；可用 `@docs/AI_CONVENTIONS.md` 引用。

## 必须遵守

1. 不展示中间思考过程；直接执行、给结论、给结果。
2. 回复尽量短；能一两句说清就不要长篇展开。
3. 少重复上下文；只补充新信息、变更点、结论。
4. 项目产出要落到对应文件；不要把应长期保留的信息只留在对话里。

## 什么时候直接回对话

- 一次性问答：原理解释、报错原因、方案比较、工具怎么用。
- 临时判断：是否合理、值不值得改、下一步建议。
- 与项目无关的问题：通用知识、闲聊、简单说明。
- 用户明确说“直接告诉我”“不用写文件”时。

## 什么时候写入文件

- 任务执行记录、验收结果、阻塞项、偏差说明。
- 会影响后续开发的约束、接口、字段、目录、依赖版本变化。
- 阶段状态、任务状态、可验证结论。
- 启动方式、联调方式、长期复用的操作说明。

## 写到哪里

- 任务进度、验收、阻塞：`docs/PHASE1_TASKS.md`
- 系统约束、接口、依赖、目录、架构：`docs/SYSTEM_SPEC.md`
- 阶段目标与完成口径：`docs/GOALS.md`
- 仅属于当前代码的局部说明：代码注释或邻近 README

## 输出偏好

- 多步任务：先做事，再更新文档，最后用 1～2 句总结改了什么。
- 出错时：优先直接修；修不了再把阻塞写入对应文档。
- 除非用户要求，不在对话里重复粘贴文档已有大段内容。
