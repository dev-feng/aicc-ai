# AI 协作约定（跨 IDE 通用）

在 Cursor、Codex、Copilot 或其它 AI 助手中，**请先阅读并遵守本约定**（可用 `@docs/AI_CONVENTIONS.md` 引用）。

## 必须遵守

1. **不展示中间思考过程**：不要输出「让我先…」「我来检查…」等步骤说明，直接执行并给出结果或写入文件。
2. **结果落文件**：任务结论、执行记录、验收更新等写入对应文档（如 PHASE1_TASKS.md、Spec），不在对话里长段复述。
3. **回复尽量短**：能一句话说清的不要多段；需要细节时用列表或表格，少用长段落。
4. **少重复上下文**：不重复用户已提供的信息或文档中已有的内容，只补充新信息或变更点。

## 建议

- 多步任务：用工具完成并更新文档，最后用 1～2 句总结「做了哪几件事、改动了哪些文件」。
- 出错时：直接修或写进执行记录的「偏差/阻塞」，不展开分析过程除非用户问。
